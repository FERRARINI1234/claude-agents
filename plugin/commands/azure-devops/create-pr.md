# Create Pull Request — Azure DevOps

Analise as alteracoes no repositorio e crie um Pull Request no **Azure DevOps**
seguindo as convencoes de commits semanticos do projeto.

## Instrucoes

### 1. Identificar alteracoes

```bash
git status
git log origin/master..HEAD --oneline   # commits que serao incluidos no PR
git diff origin/master...HEAD           # diff completo em relacao a master
```

### 2. Classificar o tipo de alteracao

| Prefixo     | Uso                                               |
|-------------|---------------------------------------------------|
| `feat:`     | Nova funcionalidade                               |
| `fix:`      | Correcao de bug                                   |
| `refactor:` | Refatoracao sem mudar comportamento               |
| `docs:`     | Alteracoes em documentacao                        |
| `chore:`    | Tarefas de manutencao (configs, deps, CI)         |
| `test:`     | Adicao ou modificacao de testes                   |
| `style:`    | Formatacao, espacos, ponto-virgula                |
| `perf:`     | Melhorias de performance                          |
| `ci:`       | Alteracoes em CI/CD                               |

### 3. Commit (se houver alteracoes pendentes)

```bash
git add <arquivos especificos>
git commit -m "<tipo>: <descricao curta>

<descricao detalhada opcional>"
```

### 4. Push

```bash
git push -u origin <branch-atual>
```

### 5. Criar o PR via Azure DevOps CLI

Extraia org, projeto e repositorio da URL do remote:

```bash
# Remote: https://credicoamo.visualstudio.com/GEFIP/_git/fabric
# Organizacao : credicoamo
# Projeto     : GEFIP
# Repositorio : fabric
```

Instale a extensao azure-devops se necessario:

```bash
az extension add --name azure-devops --only-show-errors
```

Configure a organizacao padrao (uma vez):

```bash
az devops configure --defaults organization=https://credicoamo.visualstudio.com project=GEFIP
```

Crie o PR apontando para `master`:

```bash
az repos pr create \
  --title "<tipo>: <descricao>" \
  --description "$(cat <<'EOF'
## Resumo
- <bullet points das alteracoes>

## Tipo de Alteracao
- [ ] feat: Nova funcionalidade
- [ ] fix: Correcao de bug
- [ ] refactor: Refatoracao
- [ ] docs: Documentacao
- [ ] chore: Manutencao / CI

## Plano de Testes
- <como validar as alteracoes>
EOF
)" \
  --source-branch <branch-atual> \
  --target-branch master \
  --repository fabric \
  --open
```

> **Autenticacao:** O `az repos pr create` requer login ativo.
> Use `az login` para autenticacao interativa, ou defina a variavel de ambiente
> `AZURE_DEVOPS_EXT_PAT` com um Personal Access Token com escopo `Code (Read & Write)`.

### 6. Fallback — URL direta

Se o CLI nao estiver disponivel, construa a URL e abra no navegador:

```
https://credicoamo.visualstudio.com/GEFIP/_git/fabric/pullrequestcreate?sourceRef=<branch>&targetRef=master
```

Substitua `<branch>` pelo nome da branch atual (`git branch --show-current`).

## Fluxo completo — exemplo

```bash
# 1. Ver o que sera incluido no PR
git log origin/master..HEAD --oneline
git diff origin/master...HEAD --stat

# 2. Commit semantico (se necessario)
git add CICD/sync_workspace_from_git.py docs/adr/ADR-0001.md
git commit -m "chore: desabilita sync Git automatico — ADR-0001"

# 3. Push
git push -u origin develop

# 4. Criar PR
az devops configure --defaults organization=https://credicoamo.visualstudio.com project=GEFIP
az repos pr create \
  --title "chore: desabilita sync Git automatico — ADR-0001" \
  --description "## Resumo\n- Step sync comentado na esteira\n- ADR-0001 adicionada" \
  --source-branch develop \
  --target-branch master \
  --repository fabric \
  --open
```

---

## NUNCA

- Commitar diretamente em `master` ou `develop` sem PR quando houver alteracoes criticas
- Usar `git add .` ou `git add -A` sem revisar o que sera incluido
- Commitar arquivos sensiveis (`.env`, secrets, credenciais)
- Incluir atribuicao de AI ou rodapes "Generated by"

## Notas

- Branch padrao de destino: `master`
- Branch padrao de origem: `develop` (feature branches tambem sao aceitas)
- Mensagens de commit em portugues (consistente com o projeto)
- O flag `--open` abre o PR no navegador apos criacao
- PAT minimo necessario: `Code (Read & Write)` no projeto GEFIP
