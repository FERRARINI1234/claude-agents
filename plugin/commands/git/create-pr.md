# Create Pull Request

Analise as alteracoes no repositorio e crie um Pull Request seguindo as convencoes de commits semanticos.

## Instrucoes

1. **Identificar alteracoes**
   - Execute `git status` para ver arquivos modificados, adicionados ou removidos
   - Execute `git diff` para entender o conteudo das alteracoes

2. **Classificar o tipo de alteracao**
   Determine o prefixo semantico baseado nas mudancas:

   | Prefixo    | Uso                                              |
   |------------|--------------------------------------------------|
   | `feat:`    | Nova funcionalidade                              |
   | `fix:`     | Correcao de bug                                  |
   | `refactor:`| Refatoracao sem mudar comportamento              |
   | `docs:`    | Alteracoes em documentacao                       |
   | `chore:`   | Tarefas de manutencao (configs, deps)            |
   | `test:`    | Adicao ou modificacao de testes                  |
   | `style:`   | Formatacao, espacos, ponto-virgula               |
   | `perf:`    | Melhorias de performance                         |
   | `ci:`      | Alteracoes em CI/CD                              |

3. **Fazer commit**
   - Adicione os arquivos relevantes com `git add <arquivos>`
   - Crie o commit com mensagem semantica:
   ```
   <tipo>: <descricao curta>

   <descricao detalhada opcional>

   Co-Authored-By: Claude <noreply@anthropic.com>
   ```

4. **Criar branch (se necessario)**
   - Se estiver na main/master, crie uma branch descritiva:
   ```bash
   git checkout -b feat/descricao-curta
   # ou
   git checkout -b fix/descricao-do-bug
   ```

5. **Push e criar PR**
   - Execute `git push -u origin <branch>`
   - Use `gh pr create` com titulo e descricao:
   ```bash
   gh pr create --title "<tipo>: <descricao>" --body "$(cat <<'EOF'
   ## Summary
   - <bullet points das alteracoes>

   ## Type of Change
   - [ ] feat: Nova funcionalidade
   - [ ] fix: Correcao de bug
   - [ ] refactor: Refatoracao
   - [ ] docs: Documentacao
   - [ ] chore: Manutencao

   ## Test Plan
   - <como testar as alteracoes>

   ---
   Generated by Claude Code
   EOF
   )"
   ```

## Exemplo de Fluxo

```bash
# 1. Ver alteracoes
git status
git diff

# 2. Adicionar arquivos
git add infra/src/helm-charts/airflow/values.yaml
git add docs/ARCHITECTURE.md

# 3. Commit semantico
git commit -m "feat: adiciona configuracao de recursos para Airflow workers

- Define requests e limits de CPU/memoria
- Adiciona configuracao de HPA

# 4. Push
git push -u origin feat/airflow-resources

# 5. Criar PR
gh pr create --title "feat: adiciona configuracao de recursos para Airflow workers" --body "..."
```
## NUNCA
- Realize commits diretamente na branch Main/Master
- Inclua nenhuma AI/Claude atribuição
- Adicione rodapés "Generated by"
- Use robot emojis
- Mencione automação
- Inclua Co-Authored-By por AI

## Notas

- NAO commitar arquivos sensiveis (.env, secrets, credentials)
- NAO usar `git add .` ou `git add -A` sem revisar
- Sempre verificar `git status` antes do commit
- Mensagens de commit em ingles ou portugues (consistente com o projeto)
